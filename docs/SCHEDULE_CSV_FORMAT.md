# Формат Google Таблицы для расписания хакатона

Поле **schedule_sheet_url** в модели `Hackathon` должно указывать на таблицу, доступную для чтения по ссылке (достаточно «Все, у кого есть ссылка — могут просматривать»).

## Лист

Используется **первый лист** (или лист с `gid` из URL, если он указан в ссылке).

Первая строка — **заголовки колонок**. Дальше — строки событий.

## Обязательные / опциональные колонки

Система ищет колонки по **имени** (без учёта регистра, пробелы нормализуются в `_`).

| Назначение | Допустимые имена заголовка |
|------------|------------------------------|
| **Время начала** (обязательно) | `start`, `start_datetime`, `datetime`, `time`, `время`, `время_начала`, `дата_время`, `начало` |
| Название | `title`, `name`, `название`, `событие`, `мероприятие` |
| Описание | `description`, `описание`, `details`, `комментарий` |
| За сколько минут напомнить | `notify_minutes_before`, `notify`, `за_минут`, `напомнить_за_минут`, `minutes_before` |

Если колонки названия нет, подставляется «Событие». Если колонки «за сколько минут» нет, используется **15** минут.

## Формат времени

Поддерживается гибкий разбор через `dateutil` (рекомендуется явный формат, например `24.05.2026 14:30` или `2026-05-24 14:30`). Время **без указания часового пояса** трактуется в зоне `TIME_ZONE` Django (по умолчанию `Europe/Moscow`).

## Как работает напоминание

Celery Beat раз в минуту запускает задачу: для каждой активной подписки скачивается CSV с Google, парсятся строки. Сообщение уходит в Telegram, когда:

`время_начала − notify_minutes_before ≤ сейчас < время_начала`

Повторно одно и то же событие одному пользователю не отправляется (учёт в `ScheduleNotificationLog`).

## Пример минимальной таблицы

| start_datetime        | title          | notify_minutes_before |
|-----------------------|----------------|------------------------|
| 2026-06-01 10:00     | Регистрация    | 30                     |
| 2026-06-01 18:00     | Демо           | 15                     |
